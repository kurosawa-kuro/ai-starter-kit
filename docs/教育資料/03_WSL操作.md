# AIによる事務処理・プログラミング自動化 WSL 操作

～ 日々の WSL（Ubuntu）操作を「画面のどこを押せばいいか」「どのコマンドを叩けばいいか」レベルで把握する ～

## 本ガイドのスコープ

本ガイドは **WSL を毎日使う上で詰まりがちな操作** を集約したリファレンスです。環境構築や AI ツールの導入は対象外（それらは [`01_環境構築ガイド.md`](./01_環境構築ガイド.md) / [`02_ハンズオン.md`](./02_ハンズオン.md) を参照）。

扱う内容:

- WSL の状態確認（起動／停止／存在）
- WSL の起動と停止
- Windows ↔ WSL 間のファイル・フォルダコピー
- WSL のフォルダを Windows エクスプローラーで開く（Reveal in Explorer）
- ホームディレクトリへの最短アクセス
- WSL のリセット（最終手段）

---

## 📌 前提知識: 「Windows 側 PowerShell」とは

本ガイドの `wsl ...` 系コマンドは **Windows 側の PowerShell** から実行します（WSL の中ではない）。

```
   PowerShell ＝ Windows に最初から入っている、
                 文字コマンドで Windows を操作するアプリ
```

**起動方法:**

   - 画面左下の「スタート」を **右クリック**
   - 「**ターミナル**」または「**Windows PowerShell**」をクリック
   - 黒い画面が開けば OK（本ガイドのコマンドは **管理者でなくて OK**。`wsl --install` のような新規導入だけ「ターミナル（管理者）」が必要）

> 💡 詳しい起動方法・管理者モードとの違いは [`01_環境構築ガイド.md`](./01_環境構築ガイド.md) の「📌 補足: PowerShell とは」を参照。

> ⚠️ **PowerShell ≠ WSL のターミナル**。
> PowerShell は `wsl ...` などの **Windows 側コマンド** を打つ場所、WSL のターミナル（VSCode から接続したもの）は `git` `python3` などの **Linux 側コマンド** を打つ場所。本ガイドでは PowerShell 用と WSL 用のコマンドを区別して記載しています（早見表 §8 も同様）。

## 1. WSL の状態確認

### 1.0 WSL の 3 つの状態（図）

WSL は次の 3 つの状態のいずれかにある。コマンドで状態間を行き来する。

```
                                        wsl --install -d Ubuntu
       ┌────────────────────┐  ◀────────────────────────────────┐
       │   ❌ 未インストール  │                                    │
       │   （何もない状態）   │  ────────────────────────────────▶│
       └────────────────────┘  wsl --unregister Ubuntu           │
                                       （💀完全消去）             │
                                                                  │
                                                                  │
                       wsl                                        │
              ┌──────────────────┐                                │
              │                  ▼                                │
       ┌────────────────────┐    ┌────────────────────┐         │
       │  💤 Stopped         │    │  🟢 Running         │         │
       │  （インストール済    │    │  （起動中・        │         │
       │    だが起動してない） │    │    使える状態）    │         │
       └────────────────────┘    └────────────────────┘         │
              ▲                  │                                │
              └──────────────────┘                                │
                wsl --shutdown                                    │
                                                                  │
       現在どの状態か？ → wsl -l -v で確認 ─────────────────────┘
```

### 1.1 一覧と状態を見る

**Windows 側 PowerShell**（管理者でなくて OK）で実行:

```powershell
wsl -l -v
```

出力例:

```
  NAME      STATE           VERSION
* Ubuntu    Running         2
```

| 列 | 意味 |
|---|---|
| `NAME` | ディストリビューションの名前（通常は `Ubuntu`）。先頭の `*` は既定 |
| `STATE` | `Running`（起動中）／`Stopped`（停止中） |
| `VERSION` | `2` であれば WSL2（推奨）、`1` だと WSL1 |

### 1.2 「存在しているか」だけ知りたい

```powershell
wsl -l
```

何も出ない／`Linux 用 Windows サブシステムには、ディストリビューションがインストールされていません。` と出る場合は **未インストール**。
[`環境構築ガイド §①`](./01_環境構築ガイド.md) からセットアップする。

### 1.3 「動いているプロセスがあるか」を知る

VSCode を開いていたり、ターミナルを開いていれば `STATE` は `Running` になる。閉じても少し時間が経つまでは `Running` のまま（WSL のアイドル時間後に自動停止）。

## 2. WSL の停止

### 2.1 全部止める（一番よく使う）

```powershell
wsl --shutdown
```

   - すべてのディストリビューションを即時停止
   - **メモリ解放** や **systemd / グループ変更を反映** したいときに使う
   - 設定変更（`/etc/wsl.conf` を書き換えたあと）の反映に必須

### 2.2 個別に止める

複数ディストリビューションがあるとき、特定のものだけ止める:

```powershell
wsl --terminate Ubuntu
```

通常は `--shutdown` で十分。

---

## 3. WSL の起動

### 3.1 ターミナルから起動

```powershell
wsl
```

既定ディストリビューション（`*` が付いているもの）が起動し、Linux のシェルに入る。

特定のディストリビューションを起動するなら:

```powershell
wsl -d Ubuntu
```

### 3.2 スタートメニューから起動

   - 画面左下「スタート」→ `Ubuntu` と検索 → `Ubuntu` をクリック
   - 黒い画面（ターミナル）が開いて、自動的にログイン状態になる

### 3.3 VSCode から起動

   - VSCode を起動
   - 画面左下の **「><」** （緑色アイコン）→ **`Connect to WSL`**
   - 接続後、`ターミナル` → `新しいターミナル`（または `Ctrl + @`）
   - これが本キットの **基本動作環境**（AI ツールはこのターミナルから叩く）

## 4. WSL ↔ Windows 間のファイル・フォルダコピー

WSL の中身は Windows の C ドライブとは別の場所に住んでいる。コピー方法は 3 通りある。

### 4.0 ファイルが住んでる「場所」のマップ（図）

```
   🪟 Windows（普段の世界）          🐧 WSL Ubuntu（プログラム世界）
   ──────────────────────────       ──────────────────────────
                                    
   📁 C:\Users\<Win名>\              📁 /home/<WSL名>/
       ├─ Desktop\                       ├─ ai-starter-kit/
       ├─ Documents\                     └─ Downloads/
       └─ Downloads\                     
                                    
        │                                  ▲
        │  双方向にアクセス可能（橋渡し）   │
        │                                  │
        ▼                                  │
   ┌──────────────────────────────────────┴────────────┐
   │                                                    │
   │  WSL からは:        /mnt/c/Users/<Win名>/...         │
   │                     （C: ドライブが /mnt/c に見える）│
   │                                                    │
   │  Windows からは:    \\wsl.localhost\Ubuntu\home\...  │
   │                     （WSL がネットワークドライブ風）  │
   │                                                    │
   └────────────────────────────────────────────────────┘

   ⚠️ <Win名> と <WSL名> は別物のことが多い:
       - <Win名>: Windows のサインインユーザー
       - <WSL名>: Ubuntu 初期セットアップ時に決めた英小文字の名前
```

### 4.0.1 どの方法で運ぶか（フローチャート）

```
   Windows ↔ WSL 間でファイルを動かしたい
                  │
                  ▼
       ┌──────────────────────────┐
       │ 数個のファイル？          │ ─── YES ───▶ A: VSCode で
       │ （手で運ぶレベル）        │              ドラッグ&ドロップ
       └──────────┬───────────────┘
                  │ NO
                  ▼
       ┌──────────────────────────┐
       │ コマンドで自動化したい？   │ ─── YES ───▶ C: cp コマンド
       │ （スクリプトに組み込む等） │             /mnt/c/ や ~/ を使う
       └──────────┬───────────────┘
                  │ NO
                  ▼
              B: Windows エクスプローラーで
                 \\wsl.localhost\Ubuntu\... を開いてコピペ
```

### 4.1 方法 A: VSCode のエクスプローラーでドラッグ&ドロップ（一番簡単）

**Windows → WSL**

   1. VSCode を **WSL 接続済み** にする（左下が `WSL: Ubuntu`）
   2. 左サイドバーのエクスプローラーで、WSL 側のフォルダ（例 `~/dev/ai-starter-kit/`）を開いておく
   3. Windows のエクスプローラーから、ファイルをそのまま VSCode のファイル一覧へ **ドラッグ&ドロップ**
   4. 「コピーするか？」と聞かれたら OK

**WSL → Windows**

   - VSCode で WSL 側のファイルを **右クリック** → `名前を付けて保存` → Windows 側のフォルダを選ぶ
   - またはファイルを開いた状態で Windows のエクスプローラーへ **ドラッグ**

### 4.2 方法 B: Windows エクスプローラーから WSL に直接アクセス

Windows エクスプローラーのアドレスバーに以下を入力すると、WSL 内のファイルがフォルダとして見える:

```
\\wsl.localhost\Ubuntu\home\<ユーザー名>\
```

   - `<ユーザー名>` は WSL のユーザー名（例 `taro`）
   - ここにファイルを **コピー&ペースト** すれば WSL に置ける
   - 逆に WSL のファイルを Windows のフォルダへコピペすれば取り出せる
   - **「クイックアクセス」にピン留め**しておくと便利

### 4.3 方法 C: WSL ターミナルで `cp` コマンド

WSL から見ると、Windows の C ドライブは `/mnt/c/` にマウントされている。

**Windows → WSL**

```bash
# 例: Windows のデスクトップから WSL のホームへコピー
cp /mnt/c/Users/<Windows ユーザー名>/Desktop/data.csv ~/

# フォルダごとコピー
cp -r /mnt/c/Users/<Windows ユーザー名>/Documents/work ~/
```

**WSL → Windows**

```bash
# 例: WSL のホームから Windows のデスクトップへコピー
cp ~/result.csv /mnt/c/Users/<Windows ユーザー名>/Desktop/
```

> 💡 `<Windows ユーザー名>` は Windows のサインインユーザー名。`echo $USER` は WSL 側の名前なので注意（別物のことが多い）。Windows 側の名前は `ls /mnt/c/Users/` で一覧確認できる。

### 4.4 どの方法を使えばいいか

| 状況 | 推奨方法 |
|---|---|
| 数個のファイルをサッと移動したい | A（VSCode ドラッグ&ドロップ）|
| たくさんのファイルをまとめて移動したい | B（エクスプローラー経由）|
| 自動化スクリプトに組み込みたい・コマンドで完結させたい | C（`cp` コマンド）|

⚠️ **避けるべき**: プロジェクト本体を `/mnt/c/...` に置いて作業すること。ファイル I/O が極端に遅くなり、Git 操作も AI ツールも遅くなる。**WSL 内ホーム（`/home/<user>/`）に置く** のが鉄則。

## 5. WSL のフォルダを Windows エクスプローラーで開く（Reveal in Explorer）

### 5.1 WSL ターミナルから

WSL のターミナルで現在のフォルダを Windows エクスプローラーで開く:

```bash
explorer.exe .
```

   - `.` は「カレントディレクトリ」を意味する
   - 別のフォルダなら `explorer.exe ~/dev/ai-starter-kit/src/office-task/data`

### 5.2 VSCode から

   - VSCode のエクスプローラー（左サイドバー）で、開きたいフォルダを **右クリック**
   - メニューから **`Reveal in File Explorer`**（または `エクスプローラーで表示`）
   - Windows のエクスプローラーが開き、そのフォルダが選択された状態になる

### 5.3 Windows エクスプローラーから直接

アドレスバーに `\\wsl.localhost\Ubuntu\home\<ユーザー名>\` と入れる（§4.2 参照）。

## 6. ホームディレクトリ・作業フォルダへの最短アクセス

> 📌 本キットの規約: 全プロジェクトは **`~/dev/`（= `/home/<ユーザー名>/dev/`）** 配下に置く。

| やりたいこと | コマンド／操作 |
|---|---|
| WSL ターミナルでホームへ | `cd ~` または `cd` |
| WSL ターミナルで作業フォルダへ | `cd ~/dev` |
| 本キットへ直行 | `cd ~/dev/ai-starter-kit` |
| 現在のフォルダのパスを知る | `pwd` |
| Windows でホームを開く | エクスプローラーのアドレスバーに `\\wsl.localhost\Ubuntu\home\<ユーザー名>` |
| Windows で `~/dev/` を開く | `\\wsl.localhost\Ubuntu\home\<ユーザー名>\dev` |
| VSCode で本キットを開く | `ファイル` → `フォルダーを開く` → `/home/<ユーザー名>/dev/ai-starter-kit/` |
| WSL ターミナルから Windows のエクスプローラーで作業フォルダを開く | `explorer.exe ~/dev` |

## 7. WSL のリセット（最終手段）

セットアップが壊れて手に負えなくなったときに使う。**Ubuntu 内のデータは完全に消える** ので注意。

```powershell
# 確認
wsl -l -v

# 完全削除（復元不可）
wsl --unregister Ubuntu

# 再インストール
wsl --install -d Ubuntu
```

   - 削除前に **重要なファイルを Windows 側にコピー**しておく（§4.3 の方法）
   - `git clone` で取り直せるリポジトリ自体は再インストール後にやり直せる
   - `~/.ssh/`、`~/.config/`、`~/.bashrc` などのカスタム設定も消える点に注意

## 8. よく使うコマンド早見表

| コマンド（PowerShell 側） | 用途 |
|---|---|
| `wsl -l -v` | 一覧 + 状態確認 |
| `wsl` | 既定ディストロ起動 |
| `wsl -d Ubuntu` | Ubuntu を起動 |
| `wsl --shutdown` | 全部停止 |
| `wsl --terminate Ubuntu` | Ubuntu のみ停止 |
| `wsl --install` | 新規インストール |
| `wsl --unregister Ubuntu` | 完全削除（注意） |
| `wsl -u root passwd <ユーザー名>` | パスワードリセット |

| コマンド（WSL 側） | 用途 |
|---|---|
| `cd ~` | ホームへ移動 |
| `pwd` | 現在地のパス表示 |
| `ls` / `ls -la` | ファイル一覧（隠しファイル含む）|
| `cp <src> <dst>` | ファイルをコピー |
| `cp -r <src_dir> <dst_dir>` | フォルダごとコピー |
| `mv <src> <dst>` | 移動・リネーム |
| `rm <file>` | ファイル削除（注意：ゴミ箱なし） |
| `rm -rf <dir>` | フォルダごと削除（**取り消し不可、慎重に**）|
| `explorer.exe .` | カレントフォルダを Windows エクスプローラーで開く |
| `code .` | カレントフォルダを VSCode で開く（WSL 接続）|

## 9. トラブルシューティング

| 症状 | 対処 |
|---|---|
| `wsl --shutdown` してもすぐに `Running` に戻る | VSCode が WSL に接続中。VSCode を閉じてから再度 `wsl --shutdown` |
| Windows エクスプローラーで `\\wsl.localhost\Ubuntu\` が開けない | WSL が起動していない。一度 `wsl` で起動してから再アクセス |
| `cp` で `Permission denied` | コピー先が書き込めない場所。`sudo cp` を試す（要パスワード）。または `~/` 配下に置く |
| `/mnt/c/...` で作業すると激遅 | WSL の仕様。**ファイルは `~/` 配下に置く**。`cd ~` してから作業する |
| VSCode の WSL 接続が切れる | 左下「><」→ `Connect to WSL` で再接続。ダメなら `wsl --shutdown` → VSCode を再起動 |
| Ubuntu のパスワードを忘れた | PowerShell で `wsl -u root passwd <ユーザー名>` を実行してパスワードリセット |
| `explorer.exe .` で「ドライブが見つかりません」 | エクスプローラーが古い表示。アドレスバーに `\\wsl.localhost\Ubuntu\` を貼ってアクセス |
