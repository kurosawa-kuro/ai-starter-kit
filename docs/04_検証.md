# 04. 検証

`src/scripts/*.sh` を実行した後の **動作確認手順**。各シェルの完了後にここをそのまま叩いて、想定どおりに動いているかを判定する。

すべての操作は **WSL 接続済みの VSCode ターミナル**（または素の WSL ターミナル）で実行する想定。

---

## 1. 検証ゲートの考え方

| ゲート | 内容 | 失敗時の典型対処 |
|---|---|---|
| G1 | コマンドが PATH に存在し、`--version` が返る | ターミナルを開き直す（環境変数の反映漏れ） |
| G2 | 簡単なサンプル実行が成功する | apt の更新失敗・ネットワーク・権限の確認 |
| G3 | 認証・サービス連携（Anthropic / OpenAI / docker daemon）が成立する | 初回ログイン、`wsl --shutdown`、`docker` グループ反映 |

各シェルは G1 までは shell 内で `--version` 表示まで行うが、G2・G3 は手動確認。

---

## 2. `setup-python.sh`（Python 3 開発環境）

### G1: コマンド存在確認

```bash
python3 --version           # Python 3.x.x
python3 -m pip --version    # pip x.y.z from .../site-packages/pip (python 3.x)
```

> 📌 本キットは仮想環境（venv / uv）を使わない方針のため、`python3-venv` は導入していません。
> `python3 -m venv` は失敗するのが正常です（`docs/01_仕様と設計.md §5.1` 参照）。

### G2: 簡単な実行

```bash
python3 -c 'print("hello")'   # → hello
```

### よくある問題

| 症状 | 原因 / 対処 |
|---|---|
| `python3: command not found` | `setup-python.sh` 未実行か apt 失敗。ログを確認して再実行 |
| `pip` 更新で `externally-managed-environment` 警告 | PEP 668 制約。`setup-python.sh` は `--user` で対処済み。追加パッケージが必要な場合も `pip install --user <pkg>` で導入する（venv は使わない方針） |

## 3. `setup-claude-code.sh`（Claude Code CLI）

### 前提

- `node` / `npm` が nvm 経由で入っていること（このシェルが導入する）

### G1: コマンド存在確認

```bash
node -v        # v20.x.x（LTS）
npm -v         # 10.x.x 以降
claude --version
```

> 反映されない場合は **一度ターミナルを開き直す**（`~/.bashrc` の nvm 読み込みが効いていない）。

### G2: 起動

```bash
claude
```

- 初回はブラウザでの Anthropic 認証が求められる（G3）
- 認証完了後、対話プロンプトに入れれば成功

### よくある問題

| 症状 | 原因 / 対処 |
|---|---|
| `claude: command not found` | nvm のロードが現セッションに反映されていない。ターミナル再起動、または `export NVM_DIR="$HOME/.nvm" && . "$NVM_DIR/nvm.sh"` |
| 起動はするがブラウザが開けない | WSL 環境でブラウザが連携できていない。`wslu` 経由などで確認、または表示された URL を Windows ブラウザに手動コピペ |

## 5. `setup-docker.sh`（Docker Engine + Compose plugin）

### 重要事項

`setup-docker.sh` の完了直後は **まだそのままでは `docker` コマンドが動かない可能性がある**。以下のどちらかが必要：

1. **Windows 側 PowerShell で `wsl --shutdown`** → WSL を起動し直し（`/etc/wsl.conf` の systemd 反映、`docker` グループ反映）
2. もしくは `newgrp docker` で現セッションのグループ反映 + `sudo systemctl start docker.service`

### G1: コマンド存在確認

```bash
docker --version           # Docker version 2x.x.x ...
docker compose version     # Docker Compose version v2.x.x
```

### G2: hello-world 実行

```bash
docker run --rm hello-world
```

- `Hello from Docker!` から始まるメッセージが出れば成功

### G3: systemd / グループ確認

```bash
systemctl is-system-running     # running または degraded（degraded でも docker は動く）
groups | tr ' ' '\n' | grep -q docker && echo "OK: docker group" || echo "NG: re-login required"
```

### よくある問題

| 症状 | 原因 / 対処 |
|---|---|
| `permission denied while trying to connect to the Docker daemon socket` | `docker` グループ反映前 / systemd 起動前。`wsl --shutdown` を Windows 側で実行して再起動 |
| `System has not been booted with systemd as init system (PID 1)` | `/etc/wsl.conf` の `[boot] systemd=true` が反映されていない。`wsl --shutdown` 必要 |
| `Cannot connect to the Docker daemon at unix:///var/run/docker.sock` | dockerd 未起動。`sudo systemctl start docker` または `wsl --shutdown` で再起動 |

## 6. `setup-vscode-extensions.sh`（VSCode 推奨拡張）

### 前提

- Windows 側 VSCode をインストール済み + WSL 拡張で WSL に接続したターミナルを開いていること（`code` コマンドが PATH にある状態）

### G1: コマンド存在確認

```bash
code --version          # バージョン番号 + commit ハッシュが返る
which code              # /mnt/wslg/... or 同等の wrapper パスが見える
```

### G2: 導入済み拡張の確認

```bash
code --list-extensions
```

最低限以下が一覧に含まれていれば成功（順不同）:

```
ms-ceintl.vscode-language-pack-ja
ms-python.python
ms-python.vscode-pylance
ms-python.debugpy
ms-vscode-remote.remote-wsl
yzhang.markdown-all-in-one
eamodio.gitlens
ms-azuretools.vscode-docker
ms-toolsai.jupyter
DBCode.dbcode
mechatroner.rainbow-csv
streetsidesoftware.code-spell-checker
```

### G3: 反映の確認

- VSCode のウィンドウを一度閉じて開き直す（または「Developer: Reload Window」）
- 日本語化が効くこと、Python ファイルを開いたときに Pylance が補完を出すことを目視で確認

### よくある問題

| 症状 | 原因 / 対処 |
|---|---|
| `code: command not found` | WSL 側で `code` が PATH に無い。VSCode の WSL 拡張が入っていないか、WSL 接続経由のターミナルではない。VSCode から WSL に接続したターミナルを開き直す |
| インストールは成功するが拡張が UI に出てこない | ウィンドウ再読み込み未実施。コマンドパレット → "Developer: Reload Window" |
| 日本語化が反映されない | コマンドパレット → "Configure Display Language" → "ja" を選んで再読み込み |

## 7. プロンプトメモが動くかの軽量確認（任意）

`src/pg/README.md` のプロンプトを実際に AI に投げて動かすミニ検証：

```bash
cd ~/dev/ai-starter-kit/src/pg
claude       # または codex
```

- README にあるプロンプト（例: 「偶数判定するコマンドラインプログラムを作成依頼 ...」）をそのままコピペして送る
- 生成された Python ファイルを `python3 <file>.py` で実行できれば、Phase 2 完了 + プロンプトの実利用まで成立している
