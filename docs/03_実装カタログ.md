## 2. `src/scripts/` シェル群

すべて **WSL Ubuntu 上で `bash <name>.sh` で実行する冪等セットアップシェル**。
共通仕様: `set -euo pipefail` / `require_linux` / `require_non_root` / 色付きログ (`log` `warn` `err`) / 完了時に日本語の次手順表示。

| ファイル | 役割 | 主な導入物 | 備考 |
|---|---|---|---|
| `setup-python.sh` | Python 3 開発環境の導入 | `python3` / `python3-pip` / `python3-dev` / `build-essential` / `ca-certificates` / `curl` / `git`（**`python3-venv` は意図的に非導入**） | apt 経由。pip は `--user` で更新（PEP 668 対策）。本キット全体の方針として **仮想環境（venv / uv）は使わない**（`docs/01_仕様と設計.md §5.1` 参照） |
| `setup-claude-code.sh` | Claude Code CLI の導入 | nvm `v0.40.1` / Node.js LTS / `@anthropic-ai/claude-code`（グローバル） | nvm 経由で Node を入れて `npm i -g`。初回起動時はブラウザで Anthropic 認証 |
| `setup-codex.sh` | OpenAI Codex CLI の導入 | nvm `v0.40.1` / Node.js LTS / `@openai/codex`（グローバル） | 同上。OpenAI サインインまたは `OPENAI_API_KEY` が必要 |
| `setup-docker.sh` | Docker Engine + Compose plugin（Docker Desktop 不使用） | `docker-ce` / `docker-ce-cli` / `containerd.io` / `docker-buildx-plugin` / `docker-compose-plugin` | Docker 公式 apt リポを登録、ユーザーを `docker` グループに追加、`/etc/wsl.conf` で systemd 有効化。完了後に Windows 側で `wsl --shutdown` 必要 |
| `setup-vscode-extensions.sh` | VSCode 推奨拡張の一括導入 | 日本語パック / Python（本体 + Pylance + debugpy）/ Remote-WSL / Markdown All in One / GitLens / Docker / Jupyter / DBCode / Rainbow CSV / Code Spell Checker | `code --install-extension --force` をループ。WSL 接続済みターミナルで `code` が PATH にある前提。`--force` で冪等 |

### 2.1 共通の設計ルール（変更時は遵守）

- `set -euo pipefail` 必須
- `require_linux` / `require_non_root` を冒頭でチェック（root 実行禁止、内部から `sudo` を呼ぶ）
- `log` / `warn` / `err` ヘルパを使う
- **冪等性**: 既導入は検出してスキップ（再実行で壊れない）
- 完了時に `print_next_steps` で日本語の次手順を表示
- apt 系は `--no-install-recommends`
- Node 系は **nvm 経由**で導入し、`npm install -g` で CLI を入れる構成を踏襲
- VSCode 拡張系は `code --install-extension --force` を使う（`--force` で既導入の更新も可能）

## 3. `src/pg/` / `src/office-task/` プロンプト集

### 3.1 `src/pg/`（Python プログラミング学習）

`README.md` に AI へ渡すためのプロンプト雛形を保管。フォーマットは `## プロンプトN: <概要>` + 箇条書き。

| 番号 | 概要 | 形式 | 主な要件 |
|---|---|---|---|
| プロンプト1 | 偶数判定 CLI | コマンドラインプログラム | Python 3、仮想環境不要、回答は日本語 |
| プロンプト2 | 偶数判定 Web アプリ | FastAPI + Jinja の Web アプリ | Python 3、仮想環境不要、UI も日本語 |

共通前提:

- **Python 3 は本リポジトリの `src/scripts/setup-python.sh` で事前インストール済み**
- 必要に応じて環境構築・起動方法も AI が日本語で案内する
- 仮想環境（uv 等）は **使わない**

`data/` は現状空（プロンプト次第で追加）。

### 3.2 `src/office-task/`（事務処理）

| 番号 | 概要 | 主な要件 |
|---|---|---|
| プロンプト1 | ファイルを中身に基づいた日本語名にリネーム | **元ファイルは絶対に変更・移動・削除しない**。コピー版を同ディレクトリに保存 |

`data/` のサンプル:

| ファイル | 内容 |
|---|---|
| `新規 データ1.csv` | 札幌市の不動産物件サンプル CSV（10 件、id / title / price / area_m2 / rooms ほか） |
| `新規 データ2.csv` | 従業員サンプル CSV（10 件、employee_id / name / department ほか） |
| `新規 テキスト ドキュメント1.txt` | 架空の社内規定サンプル（IT 企業想定、章立て構造） |
| `新規 テキスト ドキュメント2.txt` | 札幌中央区 1LDK マンションの物件紹介文 |
