# 01. 仕様と設計

本リポジトリ（**AI スターターキット**）の対象ユーザー・ゴール・全体構成。新規参加者および LLM が最初に読むことを想定。

---

## 1. 想定ユーザー

- **非 IT エンジニア**で、Windows PC で日常業務をしている方
- 基本的な PC 操作（ブラウザ・エクセル等）はできる
- プログラミング経験は不要〜入門レベル
- AI コーディングツール（Claude Code / Codex CLI）を業務に取り入れたい

---

## 2. ゴール

ユーザーが Windows PC 1 台で、以下 2 つの自動化を **同じ AI 環境** で実現できる状態にする。

| 自動化対象 | 例 |
|---|---|
| ① 事務処理の自動化 | Excel 集計、ファイル整理・リネーム、PDF 処理、メール作成、Web 情報の収集 など |
| ② プログラミングの自動化／学習 | Python スクリプトの設計・実装・修正、Git 操作、ライブラリ導入 など |

「コードが書けなくても、日本語で AI に指示すれば動く」を目指す。

---

## 3. スコープ

### 3.1 本リポジトリが扱う範囲

- **環境構築ガイド**（Windows + WSL + VSCode + Git の手動セットアップ手順書）
- **WSL 上での AI ツール／ランタイム導入シェル**（Python / Claude Code / Codex CLI / Docker）
- **学習・実作業用のプロンプトメモ**（事務処理タスク／Python プログラミング学習）
- **プロンプトと一緒に使うサンプルデータ**（CSV・テキスト等）

### 3.2 本リポジトリが扱わない範囲

- 各 AI ツール（Claude Code / Codex CLI）本体の実装
- Windows / WSL / VSCode 自体のインストーラ
- 業務固有のシステム連携（社内 API、SaaS 連携など）
- 本格的なソフトウェア開発フレームワーク（CI/CD・テスト基盤・IaC など）

## 5. 設計上の決め事

### 5.1 仮想環境（venv / uv 等）は使わない

`src/pg/` の Python 学習タスクでは、仮想環境を **意図的に使わない**。学習者がプログラム本体に集中できるようにするための判断。

### 5.2 元データは絶対に変更・移動・削除しない

`src/office-task/` の事務処理タスクで `data/` 配下を扱う場合、**元ファイルは触らず、コピーして新しい名前を付けて同ディレクトリに保存する** のが既定方針。

### 5.3 プロンプトメモはセクションごとに自己完結させる

`src/{pg,office-task}/README.md` のプロンプトメモは、**共通要件を上部に DRY 化せず**、各セクションで重複していても自己完結させる。利用者が単独でコピペして AI に渡せるようにするため。

### 5.4 セットアップシェルは冪等

`src/scripts/*.sh` は再実行しても壊れない設計（既導入はスキップ／更新のみ）。詳細は [`03_実装カタログ.md §2`](./03_実装カタログ.md#2-srcscripts-シェル群) と [`CLAUDE.md`](../CLAUDE.md) のセットアップシェル設計ルール参照。

### 5.5 言語

ドキュメント・回答・コミットメッセージはすべて **日本語**。読者は日本語話者の非エンジニアを想定。
