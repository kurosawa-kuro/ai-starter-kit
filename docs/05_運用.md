# 05. 運用

利用者が **このリポジトリを使って AI で実作業を始めるまで** の手順と、日常的な使い回しの流れ。

---

## 1. 前提

- [`./教育資料/01_環境構築ガイド.md`](./教育資料/01_環境構築ガイド.md) の **Phase 1（手動セットアップ）** が完了していること。具体的には:
  - WSL（Ubuntu）が動く
  - VSCode + WSL 拡張で WSL に接続できる
  - `git --version` / `python3 --version` が WSL 内で通る
- 初回の git clone + `setup-*.sh` 実行は **[`./教育資料/02_ハンズオン.md`](./教育資料/02_ハンズオン.md)** にも詳細手順があり、本ドキュメント `§2` と同等の内容
- 以下の手順は **VSCode から WSL 接続したターミナル**（または素の WSL ターミナル）で実行する

---

## 2. 初回セットアップ（git clone → AI 環境構築）

### STEP 1. リポジトリを clone

```bash
# 作業フォルダ ~/dev/ を用意（規約：すべての clone は ~/dev/ 配下）
mkdir -p ~/dev && cd ~/dev

# HTTPS（推奨）
git clone https://github.com/kurosawa-kuro/ai-starter-kit.git
# もしくは SSH
# git clone git@github.com:kurosawa-kuro/ai-starter-kit.git

cd ai-starter-kit                  # → ~/dev/ai-starter-kit/
```

> ⚠️ Windows 側（`/mnt/c/...`）には clone しない。ファイル I/O が遅くなる。**`~/dev/`（= `/home/<user>/dev/`）配下に置く** のが規約。

### STEP 2. Python 環境を入れる

```bash
bash src/scripts/setup-python.sh
```

- 完了後、`python3 --version` が通ること
- 詳細は [`04_検証.md §2`](./04_検証.md#2-setup-pythonshpython-3-開発環境)

### STEP 3. Claude Code を入れる

```bash
bash src/scripts/setup-claude-code.sh
```

- 完了後、**一度 WSL ターミナルを開き直す**（nvm の PATH 反映のため）
- `claude --version` が通ること
- 詳細は [`04_検証.md §3`](./04_検証.md#3-setup-claude-codeshclaude-code-cli)

### STEP 4.（任意）Codex CLI を入れる

Claude Code に加えて Codex も併用したい場合:

```bash
bash src/scripts/setup-codex.sh
```

### STEP 5.（任意）VSCode 推奨拡張を入れる

エディタ周りを一気に整えたい場合（日本語化 / Python / Markdown / GitLens / Docker / Jupyter / DBCode / Rainbow CSV など）:

```bash
bash src/scripts/setup-vscode-extensions.sh
```

- WSL 接続済みの VSCode ターミナルで実行すること（`code` コマンドが PATH にある必要がある）
- 完了後、VSCode のウィンドウを再読み込み（コマンドパレット → "Developer: Reload Window"）

### STEP 6.（任意）Docker を入れる

Docker を使うタスクをやる場合:

```bash
bash src/scripts/setup-docker.sh
# その後、Windows 側 PowerShell で wsl --shutdown して WSL を入り直す
```

詳細は [`04_検証.md §5`](./04_検証.md#5-setup-dockershdocker-engine--compose-plugin)。

### STEP 7. 動作確認

[`04_検証.md §8`](./04_検証.md#8-検証一括チェックリスト) の一括チェックリストを上から叩いて緑になることを確認。

## 3. 日常の作業フロー

### 3.1 プログラミング学習タスク（`src/pg/`）

```bash
cd ~/dev/ai-starter-kit/src/pg
claude            # または codex
```

1. `src/pg/README.md` を VSCode で開く
2. 使いたいプロンプトセクション（例: `## プロンプト1: 偶数判定 CLI`）をまるごとコピー
3. 起動した AI ツールに貼り付け、足りない情報があれば補って送信
4. AI が生成したコードをファイルに保存し、`python3 <file>.py` で実行

> プロンプトは **共通要件を含めて自己完結している** ので、1 セクションそのままコピペで成立する設計。

### 3.2 事務処理タスク（`src/office-task/`）

```bash
cd ~/dev/ai-starter-kit/src/office-task
claude
```

1. `src/office-task/README.md` から目的のプロンプトを選んでコピペ
2. 自分の業務データを `data/` 配下に置く（または別ディレクトリのパスを AI に指示）
3. AI が処理を実行（Claude Code であればファイル操作も AI が直接実施）

#### 重要な約束

- **元ファイルは変更・移動・削除しない**。コピーして新しい名前を付けて保存させる
- 結果が想定外なら、ロールバックは「コピー側を消すだけ」で済む

## 4. メンテナンス

### 4.1 セットアップシェルの再実行（Node.js / CLI のアップデート）

```bash
bash src/scripts/setup-claude-code.sh        # 既に入っているものは更新
bash src/scripts/setup-codex.sh
bash src/scripts/setup-vscode-extensions.sh  # 拡張も `--force` で最新化
```

冪等なので何度実行しても安全。

### 4.2 リポジトリ自体の更新

```bash
cd ~/dev/ai-starter-kit
git pull
```

プロンプトメモやセットアップシェルの追加・修正が反映される。

### 4.3 ログイン期限切れ等

| 症状 | 対処 |
|---|---|
| `claude` 起動時に再ログインを促される | 画面の指示に従って Anthropic 認証をやり直す |
| `codex` で 401 / Unauthorized | `OPENAI_API_KEY` を再エクスポート、または `codex` でサインインし直し |
| `docker` で `permission denied` | `wsl --shutdown` で WSL 再起動 |
