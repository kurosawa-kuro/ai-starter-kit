## このガイドの目的

非ITエンジニア（基本的なWindows操作ができる方）が、**AIコーディングツール（Claude Code、Codex CLI 等）を活用して、次の2つを自動化できるようにする**ための環境構築ガイドです。

| 自動化の対象 | 例 |
| --- | --- |
| **① 事務処理の自動化** | Excel集計、ファイル整理、PDF処理、メール作成、Web情報の収集 など |
| **② プログラミングの自動化** | Pythonスクリプトの設計・実装・修正・テスト、Git操作、ライブラリ導入 など |

この2つは **目的も使い方も別物** ですが、どちらも同じ AIツール + WSL + VSCode の環境上で実現できます。

### ゴール

   - **事務処理**を AI に頼んで自動化できる（コードが書けなくても、日本語で指示すればOK）
   - **プログラミング作業**そのものを AI に書かせて／直してもらいながら進められる
   - Windows PC1台で、両方の用途を同じ環境で扱える

### なぜ WSL や VSCode が必要なのか（手段の位置づけ）

AIツール（Claude Code・Codex CLI）も、それが動かす Python などのプログラムも、**Linux 環境で動かすのが世界標準**です。
そのため、Windows PC上で本格的にAIを使うには次の準備が必要です：

| 必要なもの | 役割 | 本ガイドでの位置づけ |
| --- | --- | --- |
| **WSL（+ Ubuntu）** | Windowsの中にLinux環境を用意する | **本ガイドで手動セットアップ**（AI動作の土台） |
| **VSCode** | コードやファイルを編集する窓口 | **本ガイドで手動セットアップ**（操作の入口） |
| **Git** | プロンプト・コード一式を `git clone` で取得 | **本ガイドで手動セットアップ**（運用フェーズの入口） |
| **Python** | AIが書くコードを実行する言語 | **存在確認のみ**（詳細は clone 後のシェルが自動構築） |
| **Claude Code / Codex CLI / nvm / Node.js / Python 3 / VSCode 推奨拡張 / Docker（任意）** | AIツール本体および動作に必要な一式 | **本ガイドの対象外**（[`ハンズオン`](./AIによる事務処理・プログラミング自動化_ハンズオン.md) で `git clone` 後、リポジトリ同梱の `src/scripts/setup-*.sh` を必要分実行して導入） |

> 💡 本ガイドで人手でやるのは、**WSL・VSCode・Git・Python確認** の最低限のみ。
> あとは [`ハンズオン`](./AIによる事務処理・プログラミング自動化_ハンズオン.md) ドキュメントの手順に従って `git clone` → 同梱シェル実行 で、Claude Code や Codex CLI を含む AI環境一式が自動構築されます。

## Windows / WSL / Ubuntu / VSCode / Python の関係

「なんでこんなに色々なものを入れるの？」を、**エクセルや普段使うアプリと比較して**説明します。

### まず大前提：Linux とは何か

> **Linux = プログラムが最も効率よく動作するように設計されたコンピュータOS。**
> 日常のオフィス作業（メール・表計算・資料作成）用ではなく、**プログラムを動かすこと自体に最適化されている**OSです。

そのため：

   - 世界中の **サーバー・クラウド・スーパーコンピュータの大半は Linux** で動いている
   - **AI・機械学習・Web開発のツールは Linux 前提** で作られているものがほとんど
   - プログラミングをするなら、Linux環境はほぼ必需品

→ だから「Windowsの中にLinuxを置く仕組み（＝WSL）」が重要になります。

### 普段の世界とプログラミングの世界の対応関係

| 普段の世界 | プログラミングの世界 | 役割 |
| --- | --- | --- |
| **Windows** | **Windows**（同じ） | PCのOS（土台） |
| **エクセル** | **VSCode** | 中身を編集するアプリ |
| **`.xlsx` ファイル** | **`.py`（Pythonコード）など** | 編集する対象のファイル |
| **エクセルの数式・関数** | **Python（プログラミング言語）** | 計算や処理をする中身 |

つまり、**「VSCodeはエクセルみたいなもの、Pythonはエクセルの関数みたいなもの」** と思ってOKです。

### Windows と Linux の違い（図1）

冒頭で説明したとおり、**WindowsもLinuxも同じ「OS」** です。違いは「**何に最適化されているか**」だけ。

```
┌─── Windows ──────────────┐     ┌─── Linux（Ubuntu）─────────┐
│                           │     │                             │
│  💻 デスクトップ画面       │     │  ⌨️  ターミナル（黒い画面）  │
│  🖱️ マウスでアイコン操作   │     │  📜 キーボードで文字コマンド │
│                           │     │                             │
│  使う人：一般ユーザー       │     │  使う人：開発者・サーバー管理者│
│  例：エクセル、Word、Edge   │     │  例：Python、Git、AIツール   │
│                           │     │                             │
│  特徴：見やすい・直感的     │     │  特徴：高速・軽量・自由度高い │
└──────────────────────────┘     └────────────────────────────┘
            ↓                                  ↓
        どちらも「OS」（パソコンを動かす基本ソフト）
            ↓
  ⭐ プログラミングの世界の「標準」は Linux 側
  ⭐ なので Windowsの中で Linux も使えると便利！
            ↓
  そのための仕組みが **WSL** です
```

**Ubuntu** は数あるLinuxの種類（ディストリビューション）のひとつで、初心者向けで一番ポピュラー。WSLで標準的に使われます。

### では「WSL」って何？

> エクセルなら計算機能はエクセル自体に入っていますが、
> プログラミングの世界では **「計算するための環境（Linux/Ubuntu）」を別に用意** する必要があります。

なぜかというと：

   - プログラミングのツール（Python・Git・AIツールなど）の多くは **Linux（Ubuntu）の上で動く前提** で作られている
   - Windowsに直接入れると相性問題でうまく動かないことが多い
   - そこで、**Windowsの中に「小さなLinux PC」を一台置く** という仕組みが用意された
   - それが **WSL（Windows Subsystem for Linux）** という、Microsoftが公式に提供する仕組み

```

│   │              │  │              │  │  （Windowsの中の    │  │

│   │              │  │   WSL拡張で接続 │   ├────────────────┐│  │

### 操作の流れ（あなたが何かをした時に何が起きるか）（図2）

「VSCodeで `python hello.py` を実行する」という1回の操作で、裏では次のことが起きています。

```
   👤 あなた
    │
    │ ① キーボード・マウスで操作
    ▼
 ┌────────────────────────────────────────────┐
 │  💻 Windows画面の中の VSCode                 │
 │                                              │
 │   ┌────────────────────────────────┐       │
 │   │ 📝 エディタ                     │       │

 │   │   $ python hello.py            │ ← ②入力 │
 │   └────────────────────────────────┘       │
 └─────────────────────┬──────────────────────┘
                       │
                       │ ③ WSL拡張機能が
                       │   Windows ↔ Ubuntu を橋渡し
                       ▼
 ┌────────────────────────────────────────────┐
 │  🐧 WSL の中の Ubuntu（Linux）              │
 │                                              │
 │   ┌────────────────────────────────┐       │

   - **見えているのは「VSCodeの画面」だけ**（だからエクセルを使うのと同じ感覚で操作できる）
   - **実際に動いているのは Ubuntu（Linux）側**
   - 両者を **WSL拡張機能** が自動で橋渡ししてくれるので、ユーザーは意識しなくてOK

### よくある疑問

**Q. なぜVSCodeから直接Windows側のPythonを使わないの？**
A. Windowsに直接Pythonを入れると、ネット上のサンプルコードや手順書（多くがLinux向け）がそのまま動かないことが多く、トラブルが起きやすいです。Ubuntuに入れれば「世界中の人と同じ環境」になるので情報が見つけやすく、AIツール（Claude Code・Codex CLI）も正常に動きます。

**Q. VSCodeはどこにインストールするの？**
A. **VSCode本体はWindowsアプリ** として普通にインストールします（エクセルと同じ）。それに「WSL拡張機能」を追加することで、Ubuntu側のファイルも編集できるようになります。

**Q. ファイルはどこに保存される？**
A. プログラミング作業のファイルは **Ubuntu内（`/home/ユーザー名/`）** に保存します。エクセルでいう「ドキュメントフォルダ」のような場所がUbuntu内にある、というイメージです。

> 💡 **覚え方**：
> - **Windows = PC本体**
> - **VSCode = エクセルのようなアプリ（窓口）**
> - **WSL + Ubuntu = Windowsの中に置いた“もう一台のLinux PC”（作業場）**
> - **Python など = その作業場で使う道具**

## ① WSL（Ubuntu）のインストール

### 前提条件
- Windows 10 バージョン2004以降（ビルド19041以降）または Windows 11
- 管理者権限のあるアカウントでログインしていること

### 手順

**1. PowerShellを「管理者として」起動する**
   - 画面左下の「スタート」ボタンを右クリック
   - メニューの中から **「ターミナル（管理者）」** または **「Windows PowerShell（管理者）」** を選択
   - 「このアプリがデバイスに変更を加えることを許可しますか？」と聞かれたら **「はい」** をクリック

**2. 以下のコマンドを入力してEnterキーを押す**

```powershell
wsl --install
```

   - 自動的にUbuntuがインストールされます
   - 完了したら **PCを再起動** してください

**3. 再起動後、Ubuntuが自動起動する**
   - 黒い画面が開き、しばらく待つとユーザー名とパスワードの入力を求められます
   - **ユーザー名**：好きな英小文字（例：`taro`）
   - **パスワード**：好きな文字列（入力しても画面に表示されませんが、ちゃんと入力されています）
   - パスワードは2回入力します

**4. インストールが成功したか確認**
   - PowerShell（管理者じゃなくてOK）を開いて以下を実行

```powershell
wsl -l -v
```

   - `Ubuntu` が表示され、`STATE` が `Running` または `Stopped`、`VERSION` が `2` になっていれば成功

### 主なWSLコマンド

| コマンド | 用途 | 主な使用場面 | 注意点 |
| --- | --- | --- | --- |
| `wsl --install` | WSLの新規インストール | 初回セットアップ時 | 管理者PowerShellで実行 |
| `wsl` | 既定ディストリビューション起動 | 通常起動 | 既定設定が必要 |
| `wsl -l -v` | インストール済み一覧＋状態表示 | 状態確認・WSLバージョン確認 | `--list --verbose`の短縮形 |
| `wsl --shutdown` | WSL全体停止 | メモリ解放・再起動前 | 全ディストリビューション停止 |
| `wsl --unregister Ubuntu-22.04` | 指定ディストリビューション削除 | 環境リセット時 | データ完全削除（復元不可） |

## ② VSCodeのインストール

### 手順

**1. ダウンロードページを開く**
   - ブラウザで以下のURLにアクセス
   - https://code.visualstudio.com/

**2. ダウンロードボタンをクリック**
   - 「Download for Windows」と書かれた青いボタンをクリック
   - インストーラ（`.exe`ファイル）がダウンロードされます

**3. インストーラを実行**
   - ダウンロードした `VSCodeUserSetup-x64-x.x.x.exe` をダブルクリック
   - 「使用許諾契約書の同意」→ **「同意する」** を選択
   - インストール先はそのままで **「次へ」**
   - 「追加タスクの選択」では以下にチェックを入れることを推奨
     - ☑ **「PATHへの追加」**（既定でチェック済み）
     - ☑ 「エクスプローラーのファイルコンテキストメニューに〜」
     - ☑ 「エクスプローラーのディレクトリコンテキストメニューに〜」
   - 「インストール」をクリック → 完了

**4. VSCodeを起動**
   - スタートメニューから「Visual Studio Code」を検索して起動
   - 初回起動時にテーマや言語の選択画面が出ます（後から変更可能）

### 日本語化（任意）

   - 左サイドバーの **「拡張機能」アイコン**（四角が4つのマーク）をクリック
   - 検索欄に `Japanese` と入力
   - 「Japanese Language Pack for Visual Studio Code」を **Install**
   - 右下に「Restart」ボタンが出るのでクリックして再起動

> 💡 日本語化を含む VSCode 推奨拡張は、ハンズオン側で `setup-vscode-extensions.sh` により一括導入もできます（Phase 2）。今ここで日本語化だけ手動で入れておいても、後でシェルを実行しても、どちらでも構いません。

## ④ VSCodeからWSL接続確認

### 手順

**1. VSCodeを起動した状態で、画面左下の緑色「><」アイコンをクリック**

**2. 上部に表示されるメニューから「Connect to WSL」を選択**
   - 「WSLへの接続」と表示されることもあります

**3. 接続中の表示を待つ**
   - VSCodeが自動的に再起動されます
   - 左下のアイコン表示が **「WSL: Ubuntu」** に変われば接続成功

**4. ターミナルで動作確認**
   - メニューバー：「ターミナル」 → 「新しいターミナル」
   - または **`Ctrl + @`** （Ctrl + Shift + @ の場合あり）
   - 開いたターミナルで以下を実行

```bash
uname -a
```

   - `Linux ...` で始まる文字列が表示されればWSL内で動作しています

**5. ファイル操作の確認（任意）**
   - 「ファイル」 → 「フォルダーを開く」
   - 上部のパス欄に `/home/ユーザー名/` と入力してEnter
   - WSL内のホームディレクトリが開けば、本格的に開発を始められます

## ⑤ Git（WSL内・clone用）の設定 【必須】

ハンズオンの最初に **セットアップシェル・プロンプトメモ・サンプルデータ一式を `git clone` で取得する** ため、Git のセットアップは必須です。

WSLに接続した状態のVSCodeターミナルで実行します。

**1. インストール確認**

```bash
git --version
```

   - 通常は最初からインストールされています。バージョンが出ればOK
   - 出ない場合は `sudo apt update && sudo apt install -y git` でインストール

**2. 初期設定（最初の1回のみ）**

```bash
git config --global user.name "あなたの名前"
git config --global user.email "あなたのメールアドレス"
```

   - 自分が誰としてコミットするかをGitに教える設定です
   - clone するだけなら名前・メアドの内容は仮でも動きますが、後で履歴に残るので実名を推奨

**3. 動作確認（公開リポジトリでテストclone）**

```bash
cd ~
git clone https://github.com/octocat/Hello-World.git
ls
```

   - `Hello-World` フォルダが作成されていればクローン成功
   - 確認できたら `rm -rf ~/Hello-World` で削除しておいてOK

**4. プライベートリポジトリ（社内配布など）にアクセスする場合**

   - 配布元から **HTTPSのURL + アクセストークン** または **SSH鍵** の方法を指定されます
   - HTTPSの場合：clone時にユーザー名とトークン（パスワード代わり）を求められる
   - SSHの場合：別途 SSH鍵を生成して GitHub等に登録する必要があります（ハンズオン側のトラブルシューティング参照）

## ⑥ Python（WSL内）の存在確認 【必須】

> 📌 **このガイドでは Python は「存在確認」のみ行います。**
> Python 3 本体および開発用 apt パッケージ（`python3-pip` / `python3-venv` / `build-essential` 等）の追加導入は、**ハンズオン側で `git clone` するリポジトリ同梱の `src/scripts/setup-python.sh`** が自動で実施します（このリポジトリの学習タスクでは仮想環境（venv／uv 等）は意図的に使いません）。

WSLに接続した状態のVSCodeターミナルで実行します。

```bash
python3 --version
```

   - `Python 3.x.x` と表示されればOK（Ubuntuには通常プリインストール）
   - 万一「コマンドが見つかりません」が出た場合のみ、次を実行：

```bash
sudo apt update
sudo apt install -y python3
```

これで⑥は完了です。本ガイドのゴール到達です 🎉

## トラブルシューティング（環境構築フェーズ）

| 症状 | 対処法 |
| --- | --- |
| `wsl --install` で「ヘルプテキスト」が出る | すでにWSLがインストール済み。`wsl --list --online` で利用可能なディストロを確認し、`wsl --install -d Ubuntu` を実行 |
| インストールが0%で止まる | `wsl --install --web-download -d Ubuntu` でやり直し |
| VSCodeの左下に「><」が出ない | WSL拡張機能がインストールできていない。③をやり直す |
| WSL接続後にコマンドが動かない | 一度 `wsl --shutdown` してから、VSCodeを再起動して接続し直す |
| Ubuntuのパスワードを忘れた | PowerShellで `wsl -u root passwd ユーザー名` でリセット可能 |
| `git --version` が出ない | `sudo apt update && sudo apt install -y git` でインストール |
| `python3 --version` が出ない | `sudo apt update && sudo apt install -y python3` でインストール |

> 💡 `git clone` 以降や `setup-*.sh` 実行時のトラブルは **[`ハンズオン`](./AIによる事務処理・プログラミング自動化_ハンズオン.md)** ドキュメント末尾のトラブルシューティングを参照してください。
